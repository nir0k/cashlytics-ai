Purpose: These actions touch financial records and cross-reference accounts and categories. DATA-10 (FK validation) is critical here — without verifying account ownership, a user could attach expenses to another user's account by guessing the UUID.
Output: Five action files fully refactored. Every CREATE validates FK ownership. Every SELECT/UPDATE/DELETE scoped by userId.
</objective>
