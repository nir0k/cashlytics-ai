  <action>
    Refactor src/actions/accounts-actions.ts applying these changes to every function:

    1. Replace `import { getCurrentUserId } from "@/lib/auth/user-id"` with `import { requireAuth } from "@/lib/auth/require-auth"` at the top. Also add `and` to the drizzle-orm imports.

    2. **getAccounts()**: Add requireAuth check at top. Add `eq(accounts.userId, authResult.userId)` to WHERE clause (currently has no WHERE — it returns ALL accounts). Pattern:
       ```typescript
       const authResult = await requireAuth();
       if (authResult.error) return { success: false, error: "Unauthorized" };
       // ... query with .where(eq(accounts.userId, authResult.userId))
       ```

    3. **getAccountById(id)**: Add requireAuth check. Change WHERE to `and(eq(accounts.id, id), eq(accounts.userId, authResult.userId))` — this prevents fetching another user's account by ID.

    4. **createAccount(data)**: Replace `const userId = await getCurrentUserId()` with requireAuth pattern. Keep the insert logic — just swap userId source.

    5. **updateAccount(id, data)**: Add requireAuth check. Change WHERE from `eq(accounts.id, id)` to `and(eq(accounts.id, id), eq(accounts.userId, authResult.userId))` — prevents updating another user's account.

    6. **deleteAccount(id)**: Add requireAuth check. Change WHERE from `eq(accounts.id, id)` to `and(eq(accounts.id, id), eq(accounts.userId, authResult.userId))` — prevents deleting another user's account.

    All functions must return `{ success: false, error: "Unauthorized" }` (not throw) when requireAuth fails.
    Keep all existing error messages in German unchanged.
    Keep revalidatePath calls unchanged.
