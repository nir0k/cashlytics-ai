Output: Four files refactored. Transfer FK validation enforced in both directions. Document API route secured.

    2. **getTransfers(filters?)**: Add requireAuth guard. Add `eq(transfers.userId, authResult.userId)` to the conditions array (always included, even when no filters).     3. **createTransfer(data)**: Replace getCurrentUserId with requireAuth. Add bidirectional FK validation BEFORE the transaction:
       ```typescript
       const { userId } = authResult;
       // FK Validation: BOTH accounts must belong to the authenticated user (DATA-10)
       const [sourceAccount] = await db
         .select({ id: accounts.id })
         .from(accounts)
         .where(and(eq(accounts.id, data.sourceAccountId), eq(accounts.userId, userId)))
         .limit(1);
       if (!sourceAccount) {
         return { success: false, error: "Quellkonto nicht gefunden oder kein Zugriff." };
       }
       const [targetAccount] = await db
         .select({ id: accounts.id })
         .from(accounts)
         .where(and(eq(accounts.id, data.targetAccountId), eq(accounts.userId, userId)))
         .limit(1);
       if (!targetAccount) {
         return { success: false, error: "Zielkonto nicht gefunden oder kein Zugriff." };
       }
       ```
       Then proceed with the existing db.transaction logic (add userId to the insert).

    4. **updateTransfer(id, data)**: Add requireAuth. The update uses a transaction that first fetches the old transfer. Change this initial fetch to include userId check: `and(eq(transfers.id, id), eq(transfers.userId, authResult.userId))`. If not found (null), return `{ success: false, error: "Transfer nicht gefunden." }` (currently it throws — change to return to avoid stack trace exposure).
       Also update the final `tx.update(transfers).set(data).where(eq(transfers.id, id))` to include userId: `and(eq(transfers.id, id), eq(transfers.userId, authResult.userId))`.

    5. **deleteTransfer(id)**: Add requireAuth. Same as updateTransfer — the transaction fetches the transfer first. Change to include userId: `and(eq(transfers.id, id), eq(transfers.userId, authResult.userId))`. If null, return error instead of throw. Also add userId to the final delete: `and(eq(transfers.id, id), eq(transfers.userId, authResult.userId))`.

    This file uses NextResponse, not ApiResponse. Do NOT use requireAuth (it's designed for Server Actions returning ApiResponse). Instead, use auth() directly:

    1. Add `import { auth } from "@/auth"` at the top. Remove `import { getCurrentUserId } from "@/lib/auth/user-id"`.

    2. **POST handler**: Replace `const userId = await getCurrentUserId()` with:
       ```typescript
       const session = await auth();
       if (!session?.user?.id) {
         return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
       }
       const userId = session.user.id;
       ```
       Place this auth check BEFORE the formData parsing (fail fast). Keep all validation and rate limiting logic.

    3. **GET handler**: Currently returns ALL documents with no auth. Add:
       ```typescript
       export async function GET() {
         const session = await auth();
         if (!session?.user?.id) {
           return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
         }
         try {
           const userDocuments = await db
             .select()
             .from(documents)
             .where(eq(documents.userId, session.user.id));
           return NextResponse.json(
             userDocuments.map((d) => ({
               id: d.id,
               fileName: d.fileName,
               mimeType: d.mimeType,
               size: d.size,
             }))
           );
         } catch (error) {
           logger.error("Document fetch error", "GET /api/documents", error);
           return NextResponse.json({ error: "Fehler beim Laden" }, { status: 500 });
         }
       }
       ```
       Add `eq` to the drizzle-orm imports in route.ts.
