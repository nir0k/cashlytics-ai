Rewrite `src/app/(auth)/register/page.tsx` to be an async Server Component that checks registration status before rendering.

Current file is:

```typescript
import { RegisterForm } from "@/components/organisms/register-form";

export default function RegisterPage() {
  return <RegisterForm />;
}
```

Replace with:

```typescript
import { redirect } from "next/navigation";
import { isRegistrationOpen } from "@/lib/auth/registration-mode";
import { RegisterForm } from "@/components/organisms/register-form";

export default async function RegisterPage() {
  const open = await isRegistrationOpen();
  if (!open) {
    redirect("/login");
  }
  return <RegisterForm />;
}
```

Important:

- `redirect()` from `next/navigation` throws internally (NEXT_REDIRECT). Place it outside any try/catch — consistent with established project decision [Phase 04-01]: "redirect() placed after try/catch block".
- The function must be `async` to await `isRegistrationOpen()`.

Insert the following block between AUTH_SECRET= and the Production Tips comment:

```
# --------------------------------------------
# Registration Mode (Required for production)
# --------------------------------------------
# Controls whether new users can register.
#
# SINGLE_USER_MODE=true  — Registration is open only for the first user.
#   Once one user exists, /register redirects to /login and registerAction
#   returns an error. Intended for personal/self-hosted single-person deployments.
#
# SINGLE_USER_MODE=false (or omit) — Open registration. Any visitor can
#   create an account. Intended for shared/family/team deployments.
#
# WARNING: "false" must be the exact string — not omitting the variable.
#   If set to "false", registration remains open. If omitted, same result.
SINGLE_USER_MODE=true

# Email address of the owner in single-user mode.
# Used during data migration (see: backfill script) to assign existing
# records to this user. Must match the email used when first registering.
# Not used as a registration whitelist — any email can register as the first user.
SINGLE_USER_EMAIL=you@example.com
```

Preserve all existing content exactly. Only add the new block.
