<task type="auto">
  <name>Create email transporter module with lazy singleton</name>
  <files>src/lib/email/transporter.ts, src/lib/email/types.ts, src/lib/email/index.ts</files>
  <action>
Create the email module following the lazy singleton pattern from src/lib/db/index.ts:

1. **src/lib/email/types.ts** - Define SendEmailOptions interface:

```typescript
export interface SendEmailOptions {
  to: string;
  subject: string;
  text: string;
  html: string;
}

export interface SendEmailResult {
  success: boolean;
  error?: "SMTP_NOT_CONFIGURED" | "TRANSPORTER_UNAVAILABLE" | "SEND_FAILED";
}
```

2. **src/lib/email/transporter.ts** - Create lazy singleton transporter:

- Import nodemailer and logger from @/lib/logger
- Define SmtpConfig interface (host, port, user, pass, from)
- Create `getSmtpConfig()` function that reads from process.env (SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, SMTP_FROM)
- Return null if any required var missing
- Create cached `transporter` variable and `isConfigured` flag
- Export `isEmailConfigured()` function that checks config lazily
- Export `getTransporter()` function that creates transporter on first access with:
  - `secure: config.port === 465` (auto-detect TLS mode)
  - Warning log if SMTP not configured
- Export `sendEmail()` function that:
  - Returns early with `{ success: false, error: "SMTP_NOT_CONFIGURED" }` if not configured
  - Sends email using transporter.sendMail()
  - Logs success/failure
  - Returns `{ success: true }` or `{ success: false, error: "SEND_FAILED" }`

DO NOT call transporter.verify() on startup - let first send fail naturally.
DO NOT throw errors - always return result object for graceful degradation.

3. **src/lib/email/index.ts** - Public API:

```typescript
export { sendEmail, isEmailConfigured } from "./transporter";
export type { SendEmailOptions, SendEmailResult } from "./types";
```
