1. `TOKEN_EXPIRY_MS` constant: `60 * 60 * 1000` (1 hour in milliseconds)

2. `generateResetToken()`: Returns `{ raw: string; hash: string }`
   - Use `crypto.randomBytes(32)` to generate 256-bit token
   - Convert to hex string: `.toString('hex')` (produces 64-char string)
   - Hash the raw token with SHA-256: `crypto.createHash('sha256').update(raw).digest('hex')`
   - Return both raw token (for email) and hash (for DB storage)

3. `hashToken(raw: string)`: Returns `string`
   - Takes a raw token string and returns its SHA-256 hash
   - Use same hashing logic: `crypto.createHash('sha256').update(raw).digest('hex')`
   - Used when validating tokens from reset links

Import from `node:crypto` (Node.js built-in, no package needed).

DO NOT:

- Store raw tokens anywhere (they go in email links only)
- Use Math.random(), uuid, or nanoid (insufficient entropy)
- Use MD5 or other weak hash algorithms
