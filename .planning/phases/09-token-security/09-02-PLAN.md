Extend `src/lib/auth/reset-token.ts` with the following async functions:

1. `createResetToken(userId: string)`: Returns `Promise<string>`
   - Call `generateResetToken()` to get raw token and hash
   - Calculate `expiresAt`: `new Date(Date.now() + TOKEN_EXPIRY_MS)`
   - Insert into `passwordResetTokens` table: `{ tokenHash: hash, userId, expiresAt }`
   - Return the **raw token** (for email link), NOT the hash

2. `validateResetToken(rawToken: string)`: Returns `Promise<{ valid: true; userId: string; tokenId: string } | { valid: false }>`
   - Hash the incoming raw token with `hashToken()`
   - Single DB query with ALL conditions combined using `and()`:
     - `eq(passwordResetTokens.tokenHash, tokenHash)`
     - `gt(passwordResetTokens.expiresAt, new Date())` (not expired)
     - `isNull(passwordResetTokens.usedAt)` (not already used)
   - Return `{ valid: false }` if no match (single error message, no distinction between not-found/expired/used)
   - Return `{ valid: true, userId, tokenId }` if valid

3. `consumeResetToken(tokenId: string)`: Returns `Promise<void>`
   - Update token record: `set({ usedAt: new Date() })`
   - Where: `eq(passwordResetTokens.id, tokenId)`

4. `invalidateUserTokens(userId: string)`: Returns `Promise<void>`
   - Update ALL user's unused tokens: `set({ usedAt: new Date() })`
   - Where: `and(eq(passwordResetTokens.userId, userId), isNull(passwordResetTokens.usedAt))`
   - Called after successful password reset to invalidate any other pending reset links

Imports needed from drizzle-orm: `eq`, `and`, `gt`, `isNull`

DO NOT:

- Return different error messages for expired vs invalid (prevents oracle attacks)
