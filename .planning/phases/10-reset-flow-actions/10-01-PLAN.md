**forgotPasswordAction implementation:**

1. Extract email from formData
2. Validate email format with zod (z.string().email()) - return error ONLY for format issues
3. Look up user by email (select id, email from users where email = input limit 1)
4. If user exists AND isEmailConfigured():
   - Generate token: `const rawToken = await createResetToken(user.id)`
   - Build reset URL: `${process.env.APP_URL}/reset-password?token=${rawToken}`
   - Render email: `const { html, text, subject } = await renderResetPasswordEmail(resetUrl)`
   - Send email: `await sendEmail({ to: user.email, subject, html, text })`
   - Wrap in try/catch, log errors with logger.error but don't reveal to user
5. ALWAYS return identical message: `{ success: true, message: "If an account exists with this email, you will receive a reset link." }`

**Critical security requirements:**

- NEVER return different messages based on user existence
- NEVER throw when SMTP not configured
- Catch and log all email errors silently
